# Dependency Management (npm)

This project uses **npm** for dependency management.\
To keep builds stable and predictable, we follow strict versioning
rules.

---

# TL:DR

Follow these rules:

- Always use `~` version prefix
- Always specify full semantic versions (`x.x.x`)
- Never edit `package-lock.json` manually
- Always use npm commands to manage dependencies
- Keep dependency updates controlled and intentional

---

# package.json

`package.json` defines:

- project metadata
- scripts
- dependencies
- development dependencies
- tooling configuration

It is the central configuration file for the project.

Example:

```json
{
    "dependencies": {
        "react": "~18.3.1"
    }
}
```

---

# Versioning Rules

## Always use `~` instead of `^`

Dependencies must use the **tilde (`~`) version prefix**.

Example:

```json
"react": "~18.3.1"
```

### Why

`~` allows **patch updates only**:

    ~18.3.1

Allowed updates:

    18.3.2
    18.3.3
    18.3.4

Not allowed:

    18.4.0
    19.0.0

This prevents **unexpected breaking changes** during installs.

Using `^` allows **minor upgrades**, which can sometimes introduce
breaking behavior in poorly versioned packages.

Example:

    ^18.3.1

Would allow:

    18.4.0
    18.5.0

Which may break builds.

Using `~` keeps updates **safe and predictable**.

---

## Always specify full versions (`x.x.x`)

Dependencies must always use **three-part semantic versions**.

Correct:

    ~1.4.2

Incorrect:

    ~1.4
    ~1
    latest
    *

### Why

Full semantic versions ensure:

- reproducible builds
- predictable installs
- clear upgrade history

It also prevents npm from resolving versions differently across
machines.

---

# Never manually edit `package-lock.json`

`package-lock.json` is **automatically generated by npm**.

It records the exact dependency tree used to install the project.

Do **not manually edit this file**.

### Why

Manual changes can cause:

- corrupted dependency trees
- installation failures
- inconsistent environments
- broken CI builds

Always modify dependencies using npm commands.

Correct way:

```bash
npm install some-package
```

or

```bash
npm uninstall some-package
```

This updates both:

- `package.json`
- `package-lock.json`

safely.

---

# Updating Dependencies

When updating packages, prefer controlled updates.

Example:

```bash
npm update
```

or

```bash
npm install package-name@~x.x.x
```

Avoid mass upgrades without testing.

Always verify:

- the app still builds
- linting passes
- tests pass (if present)

---

# Installing Dependencies

Always install using:

```bash
npm install
```

This ensures dependencies match the exact versions defined in
`package-lock.json`.
